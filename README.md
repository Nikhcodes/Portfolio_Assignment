# Portfolio Website (v0.1)

## Overview

This is a minimalist portfolio website created as a school assignment. The concept is built around a blurred, atmospheric interface inspired by cassette recorder aesthetics and analog media design. The intention is to create a calm, cinematic presentation of personal information and projects while keeping the structure simple, readable, and responsive.

This is version 0.1 of the project and is still in development, with content and personal details expected to be updated over time to better reflect my work and identity.

The live version of the project can be viewed here:
https://nikhcodes.github.io/

## Design Concept

The design is based on a “field recording” theme that blends analog visual language with a modern web layout. The interface uses soft blur effects, grain, and subtle motion to create a cassette-inspired aesthetic without sacrificing usability.

The visual direction focuses on restrained typography, atmospheric layering, and a muted color system with a custom accent color. The layout is structured to feel like a recorded sequence or reel, with sections organized as “transmissions” and “sessions.”

## Development Notes

The project was initially generated through a rapid vibecoding process and then refined manually. Several meaningful changes were made to improve structure, consistency, and usability.

These improvements include a customized color palette, functional navigation across all pages, mobile responsiveness, and the addition of custom favicon assets in both PNG and ICO formats. Internal links were also corrected and organized to ensure proper navigation between pages.

## Current Status

This is an early-stage version of the portfolio. It is fully functional and deployed, but still contains placeholder content in several sections. The layout and structure are stable, but the information is not yet final and will be updated as the project evolves.

## Structure

The project follows a simple static website structure consisting of HTML pages, CSS stylesheets, JavaScript functionality, and image assets. Each page has its own stylesheet while sharing a global base style system.

The main entry point is index.html, with additional pages for about, work, and contact. All assets are linked using relative paths to ensure compatibility with GitHub Pages deployment.

## Purpose

This project serves as both a technical exercise and a design exploration. It demonstrates foundational frontend development skills, responsive layout implementation, and the ability to refine a concept into a structured and functional web experience.
